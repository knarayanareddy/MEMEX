# Configuration Reference

> Complete reference for all MEMEX configuration options.

---

## Overview

MEMEX uses a **two-layer** configuration system:

1. **User Config** (`~/.memex/config.toml`) — User-facing settings
2. **Addenda TOML** (inside package) — Operational constants (SSOT)

The addenda files are the **Single Source of Truth**. Values are never duplicated in Python code. All loaders are in `memex/config/settings.py`.

---

## Layer 1: User Config (`~/.memex/config.toml`)

Generated by `memex init`. Editable by the user.

### `[daemon]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `data_dir` | string | `~/.memex` | Root data directory |
| `log_level` | string | `INFO` | Logging level: DEBUG, INFO, WARN, ERROR, CRITICAL |
| `api_host` | string | `127.0.0.1` | API bind host (MUST be loopback) |
| `api_port` | int | `7700` | API bind port |
| `ollama_base_url` | string | `http://127.0.0.1:11434` | Ollama server URL |

### `[embedding]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `model` | string | `nomic-embed-text` | Ollama embedding model name |
| `model_version` | string | `1.5` | Model version for tracking |
| `active_collection` | string | `memex_vectors_v1` | Active ChromaDB collection name |

### `[chat]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `model` | string | `llama3:8b` | Ollama chat model name |

### `[watcher]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `paths` | list | `["~/Documents", "~/Projects"]` | Directories to watch |
| `excluded_extensions` | list | `[".git", ".svn", ...]` | Extensions to skip |

### `[browser]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `fetch_page_content` | bool | `false` | Whether to fetch page content from URLs |
| `poll_interval_seconds` | int | `300` | How often to check browser history |

### `[terminal]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `poll_interval_seconds` | int | `120` | How often to check shell history |

### `[clipboard]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `poll_interval_seconds` | int | `30` | How often to check clipboard |

### `[workers]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `count` | int | `4` | Number of worker threads |

### `[graph]`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `extract_relations` | bool | `false` | Enable relation extraction between entities |

## Layer 2: Addenda TOML (SSOT)

These files live inside the package at `memex/config/`. They define operational constants.

### Addendum A: `retrieval_weights.toml`

```toml
[hybrid_retrieval]
vector_weight    = 0.40    # ChromaDB cosine similarity weight
keyword_weight   = 0.30    # FTS5 BM25 weight
graph_weight     = 0.20    # KuzuDB entity neighborhood weight
temporal_weight  = 0.10    # Temporal recency weight
# Invariant: vector + keyword + graph + temporal == 1.0

[temporal_decay]
lambda           = 0.005   # exp(-lambda * age_days)

[retrieval_limits]
default_top_k    = 20      # Default candidates before budgeting
max_top_k        = 50      # Hard ceiling

[relation_extraction]
confidence_threshold = 0.50  # Below this, relations are discarded
```

### Addendum B: `retention.toml`

```toml
[raw_content]
purge_after_days       = 7    # Days before raw_content is NULLed

[conversations]
retain_turns_days      = 365  # Days before turns are soft-deleted

[failed_documents]
max_retry_count        = 5    # Abandon after this many retries
retry_backoff_seconds  = [1, 2, 4, 8, 16]

[queue]
max_depth              = 500  # Hard ceiling for ingestion queue
warn_threshold         = 400  # WARN logged above this

[purge_schedule]
run_interval_minutes   = 60   # How often purge job runs

[parser]
timeout_seconds        = 30   # Parser killed after this
```

## Environment Variables

Environment variables override config.toml values:

| Variable | Overrides | Default |
|----------|-----------|---------|
| `MEMEX_DATA_DIR` | `daemon.data_dir` | `~/.memex` |
| `MEMEX_LOG_LEVEL` | `daemon.log_level` | `INFO` |
| `MEMEX_API_PORT` | `daemon.api_port` | `7700` |
| `MEMEX_OLLAMA_URL` | `daemon.ollama_base_url` | `http://127.0.0.1:11434` |

### Priority Order

```
Environment Variable > config.toml > Default
```

## Adding a New Configuration Value

1. **Determine the layer**:
   - User-facing setting → `default_config.toml` + `Settings` dataclass
   - Operational constant → Appropriate addenda TOML

2. **Add to TOML file**:
   ```toml
   # In the appropriate addenda file
   [section]
   new_key = "value"
   ```

3. **Load in Python** (if addenda):
   ```python
   config = load_relevant_addenda()
   value = config["section"]["new_key"]
   ```

4. **Add invariant test**:
   ```python
   def test_new_key_exists():
       config = load_relevant_addenda()
       assert "new_key" in config["section"]
   ```

5. **Never hardcode** — the value must always come from the TOML file.
